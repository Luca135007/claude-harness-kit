# claude-harness-kit

[English](README.md) | 繁體中文

讓較便宜的 Claude 模型以資深工程師紀律工作的規則檔。

## 核心理念

大模型昂貴且有配額限制。多數工作階段跑在較便宜、較快的模型上——而品質差距不在原始智力，而在*紀律*：把整個檔案讀進上下文而不委派、自己認證自己的工作「完成了」、對死路做法重試五次、該查證事實時卻默默用猜的。

這個 kit 把這種紀律外部化成弱模型真的能遵循的檔案：

- **具體規則，不是感覺。** 每條規則都有門檻值（「超過 3 個檔案或 400 行 → 委派」）、檢查清單，或一組正反範例。抽象建議（「小心一點」）對小模型毫無價值；可執行的判準才有用。
- **指揮官不下戰壕。** 主對話負責決策；大量閱讀、批次修改、研究與驗證交給便宜的子代理，回報「結論 + `file:line` 參照」——絕不傾倒原始輸出。
- **禁止自我驗證。** 做工作的人絕不自己檢查。驗證交給一個全新上下文的代理，它拿到的是需求與成品，而不是作者的樂觀。
- **憑證據升級，憑成功降級。** 便宜層級失敗一次 → 附上完整失敗軌跡，升一級重試。昂貴模型一旦破解了模式 → 把模式交回下層批次套用。
- **事實會腐化；先查證再信任。** 模型 ID、工具可用性、路徑——凡是規則對環境的斷言，都由稽核 skill 重新查證，不憑記憶信任。

這個 kit 是在單一工作階段中由頂級模型蒸餾而成，並刻意設計成能活得比該模型久：價值在制度本身，不在寫下它的模型。

## 內容物

```
templates/
  CLAUDE.md                 ← 常駐載入的索引（≤150 行，只放路由表）
  rules/
    model-dispatch.md       ← 委派、模型／層級選擇、升級路徑、驗證契約
    judgment-rubrics.md     ← 何時升級／何時才算真的完成／何時問使用者／何時轉向
    prompt-templates.md     ← 填空式交辦提示詞：搜尋、實作、重構、研究、審查
    maintenance.md          ← 誰可以改什麼、教訓格式、修剪門檻
    lessons.md              ← 只增不改的陷阱日誌（附一則範例）
skills/
  harness-audit/SKILL.md    ← /harness-audit：查證環境事實、抓過時規則、安全修訂、commit
```

## 安裝

1. 把 `templates/CLAUDE.md` 複製到 `~/.claude/CLAUDE.md`（若已有就合併——維持 ≤150 行；它該是索引，不是論文）。
2. 把 `templates/rules/` 複製到 `~/.claude/rules/`。
3. 把 `skills/harness-audit/` 複製到 `~/.claude/skills/harness-audit/`。
4. 打開 `~/.claude/CLAUDE.md` 填入專案索引的預留欄位；略讀每個規則檔，依喜好調整門檻值。
5. 可選但建議：把 `~/.claude/rules/` 納入版本控制，讓修改可稽核、可回溯。

接著跑一次 `/harness-audit`——它會查證規則提到的每個路徑、工具與模型 ID 在*你的*機器上是否真實存在，並標出不存在的項目。

## 運作方式

`CLAUDE.md` 是唯一常駐載入的檔案。它包含核心行為規則加一張路由表：「遇到情境 X，先 Read 規則檔 Y」。其餘一切按需載入，讓常駐上下文保持精簡、細部協定保持可維護。

維護本身就是設計的一部分：教訓以固定格式附加到 `lessons.md`、修剪在固定行數觸發、`/harness-audit` 定期重新查證環境事實。制度會無聲腐化；這一套會自我稽核。

## 誠實警語

- 數字門檻（3 檔案／400 行、2 輪重試、150 行上限）是可用的啟發值，不是量測結果。請自行調校。
- `model-dispatch.md` 裡的模型 ID 會過期。稽核 skill 正是為此存在。
- 外部化的流程修的是執行品質，修不了品味——模糊的需求與需要判斷的情境仍得靠更強的模型或人類。判準的作用是讓弱模型*認出*那些時刻並升級，這是它誠實能做到的極限。
